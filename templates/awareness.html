What is Cybersecurity?

Cybersecurity is the practice of protecting computers, networks, and data from digital attacks, unauthorized access, and damage. It covers everything from your personal phone to national infrastructure.

Key Point: Everyone is a potential target — not just large companies.
track_changes

The CIA Triad

The three core principles of information security:

  • Confidentiality — Data is only accessible to authorized people
  • Integrity — Data is accurate and has not been tampered with
  • Availability — Systems are accessible when needed
lock

Defense in Depth

Don't rely on a single security measure. Use multiple layers of protection:

  • Strong passwords + MFA
  • Antivirus + Firewall
  • Regular updates + Backups
  • User awareness + Monitoring
bolt

The Human Factor

Over 90% of cyber attacks involve human error. Attackers exploit psychology — urgency, fear, greed, and trust — to bypass technical defenses.

Golden Rule: Always pause and think before clicking, downloading, or sharing information.
Authentication

keyPassword Security

Your password is often the only thing standing between an attacker and your account.

verified_user

Strong Password Formula

  • Minimum 12 characters (16+ preferred)
  • Mix of UPPERCASE and lowercase letters
  • Include numbers (0–9)
  • Include symbols (!@#$%^&*)
  • No dictionary words or predictable patterns
  • Never reuse passwords across accounts
folder_open

Use a Password Manager

A password manager generates, stores, and autofills strong unique passwords for every account. You only need to remember one master password.

Recommended:

  • Bitwarden (free, open-source)
  • 1Password (premium)
  • KeePassXC (offline, local)
smartphone

Multi-Factor Authentication (MFA)

MFA adds a second layer of verification beyond your password. Even if your password is stolen, attackers can't log in without the second factor.

MFA Types (strongest → weakest):

  • 🔑 Hardware key (YubiKey)
  • 📲 Authenticator app (Authy, Google Auth)
  • 📧 Email code
  • 📞 SMS code (least secure)

Social Engineering

phishingRecognizing Phishing

Phishing is the #1 attack vector. Learn to spot and report it every time.

mail

Email Red Flags

  • flagSender address doesn't match display name
  • flagUrgent or threatening subject lines
  • flag"Your account has been suspended"
  • flag"You've won a prize!"
  • flagRequests for passwords or payment
  • flagUnexpected attachments (.exe, .zip)
  • flagPoor grammar and spelling
  • flagGeneric greetings ("Dear Customer")
link

Spotting Fake URLs

  • paypal.com vs paypa1.com
  • amazon.com vs amazon-secure.net
  • Subdomains: paypal.evil.com
  • HTTP (no padlock) on login pages
  • Shortened URLs hiding destinations
  • Extra dashes: micro-soft.com
Tip: Hover over any link to see its true destination before clicking.
smartphone

SMS Phishing (Smishing)

Attackers send fake SMS messages pretending to be banks, delivery services, or government agencies.

  • Never click links in unexpected texts
  • Call the company directly to verify
  • Banks never ask for PINs via text
  • Delivery companies use official apps
phone

Voice Phishing (Vishing)

Attackers call you impersonating tech support, banks, or government offices to steal information.

  • Microsoft never calls you unsolicited
  • IRS communicates by mail, not phone
  • Never give remote access to strangers
  • Hang up and call the official number
Infrastructure

wifi_lockNetwork Security

Secure your connection at home, at work, and on the go.

home

Home Network Security

  • Use WPA3 or WPA2 encryption
  • Change default router admin credentials
  • Create a separate guest WiFi network
  • Keep router firmware updated
  • Disable WPS (known vulnerabilities)
  • Review connected devices regularly
coffee

Public WiFi Risks

Public WiFi at cafés, airports, and hotels is a common attack vector for MitM attacks.

  • check_circleAlways use a VPN on public WiFi
  • check_circleStick to HTTPS sites only
  • cancelNever access banking on public WiFi
  • cancelDon't auto-connect to unknown networks
lock

VPN — Virtual Private Network

A VPN encrypts all traffic between your device and the VPN server, hiding your activity from eavesdroppers.

  • Essential on public WiFi
  • Hides your IP address
  • Choose providers with no-log policies
  • Recommended: ProtonVPN, Mullvad
local_fire_department

Firewall Basics

A firewall monitors and controls incoming/outgoing network traffic based on security rules.

  • Always keep OS firewall enabled
  • Configure application-level rules
  • Block unused ports and services
  • Review firewall logs for anomalies
Devices

smartphoneMobile Security

Your smartphone holds more personal data than any other device — protect it accordingly.

install_mobile

App Security

  • Only install apps from official stores
  • Check app permissions before installing
  • Remove apps you no longer use
  • Keep apps updated to patch vulnerabilities
  • Be suspicious of apps asking excessive permissions
lock_open

Device Locking

  • Use strong PIN (6+ digits) or biometrics
  • Enable auto-lock after 30 seconds
  • Enable full-device encryption
  • Enable remote wipe capability
  • Never use pattern unlock (easily observed)
cloud

Mobile Data & Backup

  • Enable encrypted cloud backup
  • Review which apps sync to cloud
  • Use Signal for sensitive messaging
  • Be careful with location sharing
  • Check location permissions per app
report

If Your Phone is Lost/Stolen

  • Use "Find My Device" to locate/wipe
  • Change passwords for all accounts
  • Contact bank to freeze mobile banking
  • Revoke sessions on all accounts
  • Report to police for insurance purposes
Expert Level

Take your security posture to the next level with these advanced techniques.

search

Principle of Least Privilege

Grant users and systems only the minimum permissions necessary to perform their tasks.

  • Use standard user accounts daily (not admin)
  • Remove unused user accounts
  • Audit permissions quarterly
  • Use role-based access control (RBAC)
folder_copy

Backup Strategy (3-2-1 Rule)

  • 3 copies of your data
  • 2 different storage media
  • 1 offsite/cloud location
  • Test backups regularly (restore drills)
  • Encrypt all backup data
bar_chart

Security Monitoring

  • Monitor login activity on all accounts
  • Set up breach alerts (HaveIBeenPwned)
  • Review system event logs weekly
  • Use intrusion detection systems (IDS)
  • Monitor outbound network connections
science

Security Testing

  • Run vulnerability scans on your network
  • Conduct phishing simulation tests
  • Perform regular penetration testing
  • Test your incident response plan
  • Use tools: Nmap, Wireshark, Metasploit
